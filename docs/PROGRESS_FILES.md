# 进度文件职责说明

> 本文档明确各个进度文件的职责和更新时机，避免混淆

## 📁 进度文件体系

### 1. `claude-progress.txt` - 项目级里程碑

**职责**: 记录**重大里程碑**和**项目级别**的进展

**更新时机**:
- ✅ 完成一个完整的 Milestone（如 M1、M2 Phase 1）
- ✅ 完成重要的技术决策（如选定数据库架构）
- ✅ 完成重要的功能集成（如新功能上线）
- ✅ 项目状态变更（如部署上线、测试通过数量大幅增加）

**不应记录**:
- ❌ 单个文件的修改
- ❌ 小 bug 修复
- ❌ 代码重构
- ❌ 文档更新（除非是重大文档重构）

**示例**:
```
### 2026-02-26
- M0 基础设施完成（CLAUDE.md 适配、文档清理）
- docs 目录整理完成
```

---

### 2. `progress.md` - 开发细节

**职责**: 记录**日常开发**的**进展**和**技术细节**

**更新时机**:
- ✅ 实现一个新功能
- ✅ 修复一个 bug
- ✅ 优化一个模块
- ✅ 添加一个测试
- ✅ 重构代码
- ✅ 调整配置或依赖

**不应记录**:
- ❌ 项目级别的里程碑（应该记录在 claude-progress.txt）

**示例**:
```
## 2026-02-26
- 实现 a2a-router.js 的链式调用
- 修复 chat-server.js 的 WebSocket 断线重连 bug
- 优化 ai-cli.js 的进程管理
```

---

## 🎯 更新决策树

```
完成了一项工作
    │
    ├─ 是否是完整的 Milestone？
    │   └─ 是 → 更新 claude-progress.txt
    │
    └─ 是否是代码改动？
        └─ 是 → 更新 progress.md
```

---

## 📝 实际案例

### 案例 1: 实现了 A2A 路由功能

**应该更新**:
- ✅ `progress.md` - 记录实现细节

**不应该更新**:
- ❌ `claude-progress.txt` - 这不是里程碑级别的改动

---

### 案例 2: M1 基础交互完成（召唤、状态栏、配置全部完成）

**应该更新**:
- ✅ `claude-progress.txt` - 记录里程碑完成
- ✅ `progress.md` - 记录最后的集成细节

---

### 案例 3: 修复了一个 WebSocket 断线重连的 bug

**应该更新**:
- ✅ `progress.md` - 记录 bug 修复

**不应该更新**:
- ❌ `claude-progress.txt` - 这不是里程碑级别的改动

---

### 案例 4: 重构了整个文档结构，增强了项目吸引力

**应该更新**:
- ✅ `claude-progress.txt` - 这是项目级别的重要改动

## 🤖 AI Agent 使用指南

### 完成任务后的检查清单

```
[ ] 这是否是一个完整的 Milestone？
    → 是：更新 claude-progress.txt 顶部"最近活动"

[ ] 这是否涉及代码改动？
    → 是：更新 progress.md
```

### 快速判断规则

- **大改动** → `claude-progress.txt`
- **日常开发** → `progress.md`

---

## 📊 文件对比表

| 文件 | 粒度 | 更新频率 | 内容类型 | 读者 |
|------|------|----------|----------|------|
| `claude-progress.txt` | 项目级 | 低（里程碑） | 重大进展 | 所有人 |
| `progress.md` | 模块级 | 高（每日） | 开发细节 | 开发者 |

---

## ⚠️ 常见错误

### ❌ 错误 1: 在 claude-progress.txt 记录小改动
```
### 2026-02-26
- 修复了 ai-cli.js 的一个 typo
- 调整了日志格式
```
**正确做法**: 这些应该记录在 `progress.md`

---

### ❌ 错误 2: 在 progress.md 记录里程碑
```
## 2026-02-26
- M1 完成！
```
**正确做法**: 里程碑应该记录在 `claude-progress.txt`，progress.md 只记录具体的代码改动

---

## 🎯 总结

**一句话原则**:
- **大事记** → `claude-progress.txt`
- **日常开发** → `progress.md`

**记住**: 进度文件是为了**快速定位信息**，不是为了重复记录。每个文件有自己的职责，不要混淆。
